No 1 About Project

A Unity 2d Android Mobile Mini Game which is an infinite scroller game where object pooling concept is used for infinite platform generation.
The ninja can jump, kill an enemy and can collect coins for score and high score will be maintained.

It has intro, pause, death screen and also sound effects. This is the first game that I have created and I wanted to showcase it. :)
How to Play:
The character moves automatically and there are 3 buttons.

  1. Jump [double jump] - right side
  2. Dive -left side
  3. Kunai - right side

You have to jump as many platforms as possible without landing on spikes and killing enemies.

No 2 The Process

The game is an infinte side scroller game. You have to jump on platforms, dodge spikes and also kill enemies on the way. You should also collect coins to get more score.

No 2.1Programming

  • Have created seperate scripts for each functions like coin generation, Platform generator, Object Pooling, Pickups and CoinGenerator.
    Cam Controller is for pinning the camera on the ninja so while it moves, it follows the ninja.
  • Object Pooler
    This is generic script that helps in using the already generated objects in the scene which is termed as Object Pooling
  • Platform Generator
    This is the script that takes the object from the pool and puts the objects in the random spots.
    The objects are 3 types of platforms, enemy, spikes and coins. I also made a blocker/collider to follow the player from a distance in the back to check for the platforms that got past the ninja and it deactivates the platform present in the pool.
  • Health Manager - Updates the 4 health present for the player. If the player hits the spike(1 heart) or enemy(2 hearts) hearts are lost.
    Pickups - Score is updated and also sound is played when collected.
    Spike - Calls the function in the Health Manager.
    Throw - The code for throwable is done here with the triggerCollider code and also after a certain delay the instantiated objects are destroyed.
  • Player Controller
    All the player/ninja code functionality is written here. Character animation is handled here. Score is also maintained here. When the player falls down death is also triggered here.
    Double Jump, Kunai instantiation and Dive code is written here.

No 2.2Game Design

  • Game Manager
    Game Manager has all the inital position of the ninja and all the initial parameters of the game. This is used to reset the game when the player clicks the restart button. It also has all the menu related functions.
  • Have added start screen with highscore which is stored in UserPrefs and is retrieved from it the next time the game is opened. Have also added pause and death screen. Pause screen with resume, quit and restart and death screen without resume option.
